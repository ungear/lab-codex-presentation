Variables

Avoid one letter names


						// bad
						let d = dataSource.getUser();
					

						// good
						let userProfile = dataSource.getUser();

Variables

Use is/can/have prefixes for boolean variables


						// bad
						let valid = users.filer(x => x.valid).length > 0;
						let edit = user.isAdmin || user.isEditor;
					

						// good
						let haveValidUsers = users.filer(x => x.valid).length > 0;
						let isEditingAllowed = user.isAdmin || user.isEditor;

Variables

Use postfixes to define measure units (pixels, seconds)


							// bad
							const delay = 500;
							setTimeout(() => {}, delay)
						

							// bad
							const delayMs = 500;
							setTimeout(() => {}, delayMs)

Variables

Use let and const instead of var


							// bad
							var operationStatus = "initialized"; 
							var isEditingAllowed = user.isAdmin || user.isEditor;

							operationStatus = isEditingAllowed
								? "enabled"
								: "disabled"
						

							// good
							let operationStatus = "initialized"; 
							const isEditingAllowed = user.isAdmin || user.isEditor;

							operationStatus = isEditingAllowed
								? "enabled"
								: "disabled"

Variables

Make names more detailed by using adjectives


								// bad
								const books = dataSource.getBooks();
								const someBooks = books
								  .filter(x => x.numberOfPages < 100)
								  .filter(x => x.genre === "detective");
							

								// good
								const books = dataSource.getBooks();
								const shortDetectiveBooks = books
								  .filter(x => x.numberOfPages < 100)
								  .filter(x => x.genre === "detective");
